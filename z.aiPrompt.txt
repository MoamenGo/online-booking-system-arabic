---

## 📋 معلومات المشروع الأساسية

```
اسم المشروع: booking-system
النوع: نظام حجز مواعيد أونلاين للسوق العربي
الجمهور المستهدف: عيادات، صالونات، مدارس، مطاعم عربية
الاتجاه: RTL (من اليمين لليسار)
اللغة: عربي بالكامل
```

## 🗄️ مخطط قاعدة البيانات (Prisma Schema)

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ===== نموذج العمل التجاري =====
model Business {
  id            String    @id @default(uuid())
  name          String    // اسم العيادة/الصالون/المطعم
  nameEn        String?   // اسم بالإنجليزي (اختياري)
  type          String    // clinic | salon | school | restaurant | other
  phone         String    @unique
  email         String?   @unique
  logo          String?
  description   String?
  address       String?
  city          String?
  country       String    @default("السعودية")
  currency      String    @default("SAR")
  timezone      String    @default("Asia/Riyadh")
  
  // إعدادات العمل
  workingDays   String[]  // ["الأحد", "الإثنين", ...]
  workStart     String    @default("09:00")
  workEnd       String    @default("22:00")
  slotDuration  Int       @default(30) // مدة الفاصل بالدقائق
  breakStart    String?   // وقت استراحة
  breakEnd      String?   // نهاية استراحة
  
  // إعدادات الإشعارات
  whatsappEnabled  Boolean @default(false)
  whatsappNumber   String?
  smsEnabled       Boolean @default(false)
  emailEnabled     Boolean @default(true)
  
  // إعدادات الدفع
  paymentEnabled   Boolean @default(false)
  paymentMethod    String?  // tap | moyasar | paypal
  
  // العلاقات
  owner     User      @relation("BusinessOwner")
  services  Service[]
  bookings  Booking[]
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
}

// ===== نموذج المستخدم =====
model User {
  id            String    @id @default(uuid())
  name          String
  phone         String    @unique
  email         String?   @unique
  password      String    // مشفرة بـ bcrypt
  role          String    @default("customer") // owner | customer | staff
  isActive      Boolean   @default(true)
  
  // العلاقات
  ownedBusiness Business?  @relation("BusinessOwner")
  bookings      Booking[]
  
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
}

// ===== نموذج الخدمة =====
model Service {
  id          String   @id @default(uuid())
  businessId  String
  name        String   // "كشف طبي" / "قص شعر" / "حصة تعليمية"
  description String?
  duration    Int      // المدة بالدقائق
  price       Float
  currency    String   @default("SAR")
  color       String?  // لون للعرض في التقويم
  isActive    Boolean  @default(true)
  sortOrder   Int      @default(0)
  
  // العلاقات
  business Business @relation(fields: [businessId], references: [id], onDelete: Cascade)
  bookings Booking[]
  
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  @@index([businessId])
}

// ===== نموذج الحجز =====
model Booking {
  id          String   @id @default(uuid())
  bookingNumber String @unique // رقم الحجز مثل: BK-2024-0001
  
  // معلومات الحجز
  businessId  String
  serviceId   String
  customerId  String
  staffId     String?  // في حال وجود موظفين
  
  // التاريخ والوقت
  date        DateTime // تاريخ الموعد
  startTime   String   // "10:00"
  endTime     String   // "10:30"
  
  // الحالة
  status      String   @default("pending") // pending | confirmed | cancelled | completed | no_show
  
  // تفاصيل إضافية
  customerName  String
  customerPhone String
  customerEmail String?
  notes         String?
  totalAmount   Float?
  currency      String  @default("SAR")
  
  // الإشعارات
  whatsappSent  Boolean @default(false)
  smsSent       Boolean @default(false)
  emailSent     Boolean @default(false)
  remindedAt    DateTime?
  
  // العلاقات
  business Business @relation(fields: [businessId], references: [id])
  service  Service  @relation(fields: [serviceId], references: [id])
  customer User     @relation(fields: [customerId], references: [id])
  
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  @@index([businessId])
  @@index([date])
  @@index([status])
  @@index([customerId])
}

// ===== نموذج أيام الإجازة =====
model DayOff {
  id         String   @id @default(uuid())
  businessId String
  date       DateTime
  reason     String?
  isRecurring Boolean @default(false) // يتكرر سنوياً
  
  business   Business @relation(fields: [businessId], references: [id], onDelete: Cascade)
  
  createdAt DateTime  @default(now())

  @@unique([businessId, date])
  @@index([businessId])
}

// ===== نموذج سجل الإشعارات =====
model NotificationLog {
  id         String   @id @default(uuid())
  bookingId  String?
  businessId String
  type       String   // whatsapp | sms | email
  to         String   // رقم الهاتف أو البريد
  message    String   // محتوى الرسالة
  status     String   @default("sent") // sent | failed | pending
  error      String?
  
  booking    Booking? @relation(fields: [bookingId], references: [id])
  business   Business @relation(fields: [businessId], references: [id])
  
  createdAt DateTime  @default(now())

  @@index([businessId])
  @@index([type])
}
```

### ملف server.js الرئيسي

```javascript
// backend/server.js
// قم بإنشاء سيرفر Express كامل يحتوي على:
// 1. استيراد جميع الـ dependencies: express, cors, helmet, cookie-parser, morgan
// 2. الاتصال بقاعدة البيانات عبر Prisma Client
// 3. إعداد Middlewares: cors (لسماح بالـ frontend), helmet, morgan, JSON parser
// 4. تسجيل المسارات: /api/auth, /api/bookings, /api/services, /api/business, /api/reports, /api/notifications
// 5. معالج الأخطاء العام (errorHandler middleware)
// 6. تشغيل السيرفر على PORT من الـ env أو 5000
// 7. رسالة ترحيب عربية في الكونسول عند التشغيل

// ملاحظات مهمة:
// - استخدم cors({ origin: process.env.FRONTEND_URL, credentials: true })
// - أضف مسار health check: GET /api/health يعيد { status: 'ok', timestamp }
// - استخدم app.use('/api/auth', require('./routes/auth'))
// - نفس النمط لباقي المسارات
```

### ملف routes/auth.js

```javascript
// backend/routes/auth.js
// المسارات المطلوبة:
// 
// POST /api/auth/register
//   - استقبال: { name, phone, email (اختياري), password, role? }
//   - تحقق: الاسم مطلوب، الهاتف مطلوب وفريد، كلمة المرور 6 أحرف على الأقل
//   - تشفير كلمة المرور بـ bcrypt (10 salt rounds)
//   - إنشاء مستخدم جديد
//   - إذا كان الـ role === 'owner': إنشاء Business جديد مرتبط به
//   - إنشاء JWT token (expiresIn: '7d')
//   - رد: { user: (بدون password), token }
//
// POST /api/auth/login
//   - استقبال: { phone, password } أو { email, password }
//   - البحث عن المستخدم
//   - مقارنة كلمة المرور
//   - إنشاء JWT
//   - رد: { user, token, business? }
//
// GET /api/auth/me (محمي بـ auth middleware)
//   - إرجاع بيانات المستخدم الحالي مع بيانات العمل إذا كان owner
//
// PUT /api/auth/profile (محمي)
//   - تحديث الاسم والبريد الإلكتروني
//
// PUT /api/auth/password (محمي)
//   - تغيير كلمة المرور: { currentPassword, newPassword }
```

### ملف utils/timeSlots.js

```javascript
// backend/utils/timeSlots.js
// دالة حساب الأوقات المتاحة
//
// generateTimeSlots({ workStart, workEnd, slotDuration, breakStart, breakEnd, existingBookings, serviceDuration, date })
//
// الخطوات:
// 1. إنشاء مصفوفة جميع الأوقات الممكنة من workStart إلى workEnd بفواصل slotDuration
// 2. استثناء أوقات الاستراحة (breakStart إلى breakEnd)
```

## ⚠️ ملاحظات مهمة جداً

1. **كل النصوص في الواجهة بالعربي** - لا يوجد أي نص إنجليزي يظهر للمستخدم
2. **كل التعليقات في الكود بالعربي** لسهولة الفهم والصيانة
3. **لا تستخدم أي مكتبة UI جاهزة** (مثل shadcn أو MUI) - ابنِ المكونات بـ Tailwind من الصفر
4. **لا تستخدم `any` في TypeScript** - عرّف الأنواع بشكل صحيح
5. **لا تضع بيانات حساسة في الكود** - كل شيء من env
6. **التقويم ابنيه بـ React خالص** - لا تستخدم FullCalendar أو مكتبات تقويم
7. **استخدم `fetch` أو `axios`** للتعامل مع الـ API - لا تستخدم Server Actions لاستدعاء الـ backend
8. **كل الـ API responses بهذا الشكل**: `{ success: true, data: ..., message: "رسالة عربية" }` أو `{ success: false, message: "رسالة خطأ عربية" }`
9. **التعامل مع الأخطاء**: لا تستخدم alert() أبداً - استخدم Toast notifications
10. **الأرقام**: استخدم الأرقام الإنجليزية (1, 2, 3) وليست الهندية (١، ٢، ٣) في الواجهة
